    Class IdVerification365Id

    The interface for the main entry point for the 365id ID Verification SDK.

    This interface describes how to initialize the verification flow, loading required assets, and managing the lifecycle of the verification UI.

    Example

    import { IdVerification365Id, Verification } from "@365id/id-verification";

    const verification: Verification = new IdVerification365Id();

    verification.init(
      "verification-container",
      "https://cdn.365id.com/assets"
    );

    verification.start("your-access-token", callbacks).then(
      verification.show()
    );

    • startWithTransferToken(transferToken: string): Promise<void>
      Beta

      Starts the SDK using the transfer token.

      This method is called when a transfer token is available.

      Using a transfer token to start the SDK means the registered has chosen to continue the id verification process on a second device different from where the id verification process was started.

      This happens for example when the registered has used their phone to continue the id verification process even though the verification process was started on a kiosk or PC.

      Parameters

      • transferToken: string

        A token used to securely transfer and resume the transaction on another device.

      Returns Promise<void>

      Example

      import { IdVerification365Id, Verification } from "@365id/id-verification";

      const verification: Verification = new IdVerification365Id();
      try {
        verification.init(
          "verification-container",
          "https://cdn.365id.com/assets"
        );
      } catch (error) {
        console.error("There was an initializing the id verification sdk, error: ", error)
      }
      try {
        verification.startWithTransferToken(transferToken).then(
          verification.show()
        );
      } catch (error) {
        Console.error("There was an error starting the SDK, error: ", error)
      }

      Throws

      Error Thrown if the transfer token is invalid, expired, or the SDK fails to start.

      Implementation of Verification.startWithTransferToken
